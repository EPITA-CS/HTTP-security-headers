Missing Control-Security-Policy Header

The lack of a Control-Security-Policy means that resources such as images, scripts, and videos may be loaded from any source even a malicious one. This is obviously very unsafe.

default-src 'unsafe-eval'

This property allows the use of eval() and similar methods to create code from strings. This can allow attackers to execute their own, perhaps malicious, code.

default-src 'unsafe-hashes'

It is a withdrawal directive used to specify the default content policy for most source directives. This refers to the point from which the secured document is being served, including the same URL scheme and port number.

default-src '*'

This configuration will allow resources (images, scripts, etc) to be loaded from any domain for each resource that isn't specified with sources.

default-src 'unsafe-Inline'

It allows inline resources such as script tags and styles. Inline code is not recommended as it is used to add malicious code by attackers.

default-src data:

This allows data: and URI's are to be used as a content source. This is insecure as an attacker can inject malicious data: URI's

default-src blob:

A Binary Large OBject (BLOB) is a collection of binary data stored as a single entity. Blobs are typically images, audio or other multimedia objects, though sometimes binary executable code is stored as a blob.Here URL would be the content source.Since it has collection of different file types, sometiomes there could be malicious file injected by the attacker which makes the URL insecure.

Missing HSTS Header

The lack of a HSTS header implies that the website could be loaded over HTTP instead of HTTPS. This could allow an attacker to read any information transferred between you and the website.

max-age=0

It means the browser should remember that the website must only be accessed over https for 0 seconds. This makes the HSTS header invalid and is ignored.

max-age not defined

The max-age is not defined, this is an invalid configuration of the HSTS header.

XSS Header Missing

This header is responsible for setting up a filter that protects against XSS attacks. It is not present in the headers for this site.

XSS filter off

This setting turns off the XSS filter in the browser exposing the site to XSS attacks if a vulnerability is present.

X-Frame-Options not present

This header prevents this page from loading inside another website (maybe even one that mimics this website). By not using this header this website can be loaded inside another website.

Origin-when-cross-origin

This option sends the current url (without parameters) to a new domain if it is accessed through the first domain. This will not leak sensitive information to other domains but may allow user tracking.

Unsafe-URL

This option sends the current url (with parameters) to a new domain if it is accessed through the first domain. This could leak sensitive information to other domains.

Feature-Policy *

This header allows all resources and iframes within this page to have access to system devices such as microphone and camera.

Max-age set to greater than 30 minutes

This configuration caches data for longer than 30 minutes. This is a risk as an attacker may gain access to that data before it is changed, granting them access to the site.
